## New collections

### `hotspots.jsonl`
File/module fragility, derived from `bugs.jsonl` by `memory scan-hotspots`.

**Schema** (`HOTSPOT_BODY`):
- `file_path: string` (min 1 char)
- `bug_count: number` (non-negative int)
- `recent_bug_ids: string[]` (uuid)
- `fragility_score: number` (0..1, linear: `min(1, bug_count/10)`)
- `notes?: string` (max 500 chars)

**Example entry:**
```json
{"id":"...","ts":"...","source":"cli","confidence":1,"verified":true,"stale":false,
 "context":{"files":["src/auth.ts"],"commit":"abc1234","task_summary":"hotspot scan"},
 "body":{"file_path":"src/auth.ts","bug_count":3,"recent_bug_ids":["...","...","..."],"fragility_score":0.3}}
```

### `conventions.jsonl`
Accumulated project style/conventions.

**Schema** (`CONVENTION_BODY`):
- `rule: string` (min 1) — e.g. "use `<` instead of `<=`"
- `scope: string` — glob: `src/**`, `test/**`, `all`
- `examples_good: string[]` (default [])
- `examples_bad: string[]` (default [])
- `derivation: 'user-confirmed' | 'final-reviewer-derived'`

### `module-map.jsonl`
High-level project structure, populated by `memory scan-modules`.

**Schema** (`MODULE_MAP_BODY`):
- `module: string` (min 1) — e.g. `src/api/users`
- `description: string` (min 1, max 500 chars)
- `responsibilities: string[]` (min 1)
- `key_files: string[]` (default [])
- `dependencies: string[]` (default [])

## New CLI commands

### `agentcohort memory list-runs [--limit=N] [--since=<dur>] [--json]`
Joins `start`+`end` events from `runs/INDEX.jsonl`. Output: table with run_id, ts, pipeline, tier, duration, outcome, gates fired.

```bash
agentcohort memory list-runs --limit=20 --since=7d
agentcohort memory list-runs --json | jq '.[] | select(.outcome=="aborted")'
```

### `agentcohort memory scan-modules [--root=src] [--dry-run] [--yes] [--json]`
Walks `--root`, lists top-level dirs. If `claude` CLI in PATH: shells to `claude --print --model=claude-haiku-4-5-20251001` per module to generate `MODULE_MAP_BODY` JSON, ingests. Otherwise: prints the prompt for manual paste-back.

**OpenWolf overlay**: if `.wolf/anatomy.md` exists, warns "module-map will be redundant".

### `agentcohort memory scan-hotspots [--threshold=N] [--json]`
Counts files in `bugs.jsonl` (last 30 days). For each file with ≥ threshold bugs, writes a `HOTSPOT_BODY` entry. Idempotent — re-running updates existing entries.

### `agentcohort memory compact [--collection=<name>] [--older-than=<dur>] [--keep-last=<N>] [--dry-run]`
Bookkeeping merge — replaces ≥ 10 old entries in a collection with 1 synthetic "compacted" entry preserving `merged_count + ts_range + id_range`. NEVER compacts `audit`, `verifications`, or `scratch` (audit trail preserved).

### `agentcohort memory clean --runs [--older-than=30d] [--orphans] [--dry-run]`
Reaps `runs/<id>/` directories matching: `--older-than` (oldest first) OR `--orphans` (start without end, > 1h grace period). Also strips matching INDEX events.

### `agentcohort stats [--since=<dur>] [--compare-naive] [--json]`
Aggregates `runs/INDEX.jsonl`. Token estimate via static per-agent table (see "Token estimate" section below). `--compare-naive` computes hypothetical full-pipeline cost for the same task set.

```
Runs: 47 total — Success: 37  Aborted: 6  Failed: 4
Per pipeline: /dev-flow ×12, /quick-fix ×18, /bug-audit ×7, ...
Token estimate: Actual ~$0.47   Naïve ~$1.23   Savings 62%
```

## Per-stage events in INDEX.jsonl

v0.10.0 wrote `start` + `end` per pipeline. v0.10.1 adds `stage_start` + `stage_end` per agent within the pipeline. Emitted by every bundled agent via its boot directive (auto-rebaked by `agentcohort upgrade` / `refresh-skills`).

```jsonl
{"event":"start","run_id":"...","pipeline":"dev-flow","tier":3,...}
{"event":"stage_start","run_id":"...","stage":"repo-scout","ts":"..."}
{"event":"stage_end","run_id":"...","stage":"repo-scout","outcome":"success","ts":"..."}
{"event":"stage_start","run_id":"...","stage":"solution-architect",...}
{"event":"stage_end","run_id":"...","stage":"solution-architect","outcome":"success",...}
{"event":"end","run_id":"...","outcome":"success",...}
```

## Read-time stale auto-detection

`memory read` runs `git diff --name-only <context.commit>..HEAD` per unique commit (cached per-process). Each entry gains `_effective_stale: boolean` = persisted `stale` OR git-derived stale.

Skip with `--no-stale-check` (fast reads, e.g. in CI):
```bash
agentcohort memory read decisions --no-stale-check
```

## Dispatcher memory-aware routing

Dispatcher's boot directive (auto-baked by `memoryBoot.ts` when agent name is `dispatcher`):

1. Reads `agentcohort memory list-runs --limit=50 --json`, computes Jaccard similarity (threshold 0.3) between user's task and each past task_summary.
2. Reads `agentcohort memory read hotspots --json`. Files with `fragility_score ≥ 0.5` → force architect gate ON.
3. Reads `agentcohort memory read decisions --filter=context.files=<path> --limit=5 --with-verifications` for files mentioned in the user's task.
4. Records routing reasoning via `agentcohort gate record --gate=architect --outcome=auto-skipped --proposed-content="<reasoning>"`.

## OpenWolf overlay

| OpenWolf file present | agentcohort behavior |
|---|---|
| `.wolf/anatomy.md` | `scan-modules` warns "module-map redundant" |

## Token estimate (for `stats`)

See `src/statsTable.ts`. Estimates are ±20%. Per-agent input/output tokens × tier price ($1/$5/$15 per MTok input + $5/$15/$75 output for haiku/sonnet/opus).

To override, add to `.agentcohort.json` (v0.10.2 feature — for v0.10.1, edit source).

## Migration from v0.10.0

- **Schemas additive**: no existing entry breaks. `COLLECTION_NAMES` grows from 5 → 8.
- **INDEX.jsonl**: v0.10.0 entries (no stage events) still parse via discriminated union.
- **Boot directive**: `agentcohort upgrade` (or `agentcohort refresh-skills`) re-bakes new memory section.
- **Tests**: v0.10.0 707/707 stay green; v0.10.1 adds ~100 more.
- **CLI**: 6 new commands added; existing 8 unchanged.
